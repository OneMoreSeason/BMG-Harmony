<task type="auto" tdd="true">
  <name>Task 1: Proving envelope storage</name>
  <files>server/schema.sql, server/schema.py, server/store.py</files>
  <behavior>
    - `append_proving_envelope(con, thread_id, agent_id, proved,
      not_checked, confidence)` validates thread existence.
    - Confidence is numeric and bounded from 0.0 through 1.0.
    - `proved` and `not_checked` are Markdown strings; callers may use bullets
      for multiple items.
    - Every call inserts a new row and returns
      `{envelope_id, thread_id, agent_id, timestamp, validation_status}`.
    - Update/delete triggers prevent envelope mutation.
  </behavior>
  <action>
    Add a `proving_envelopes` table with UUID7 id, thread_id, agent_id,
    timestamp, proved, not_checked, and confidence.

    Add a JSON schema validator for proving envelopes. Implement the store
    function using the same receipt/error pattern as `write_event`.
  </action>
</task>

<task type="auto" tdd="true">
  <name>Task 2: Deterministic battle-card assembly</name>
  <files>server/store.py, tests/test_phase2_battle_card.py</files>
  <behavior>
    - `get_battle_card(con, thread_id)` does not call an LLM or external memory.
    - Unknown thread raises ToolError so route mistakes are visible.
    - Card fields include at minimum:
      `thread_id`, `topic`, `state`, `latest_decision`, `open_flags`,
      `latest_event`, `unacked_count`, `dissent_count`, `envelope_count`,
      `latest_envelopes`.
    - Latest decision is derived deterministically from the latest `position`
      event when present; otherwise latest event content is used as a fallback.
    - Open flags identify unacked messages, dissent records, and missing
      proving envelopes without making governance decisions.
  </behavior>
  <action>
    Implement battle-card assembly as a store-layer read over `threads`,
    `events`, `message_acks`, and `proving_envelopes`.

    Keep the lean card compact. Do not embed full transcript dumps in
    `get_battle_card`; callers can use `read_thread` for full history.
  </action>
</task>
