## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client→harmony_server.py tools | Tool arguments arrive from agent (Claude/Codex) via MCP protocol; treated as untrusted input to be validated by store layer |
| harmony_server.py→stdout | MCP wire protocol exclusively owns stdout; any write to stdout outside FastMCP corrupts the JSON-RPC stream |
| process env→HARMONY_DB_PATH | DB path read from environment; must be an absolute path to a writable directory |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-01 | Denial of Service | harmony_server.py — stdout pollution | mitigate | logging.basicConfig(stream=sys.stderr) at module top; no print() calls without file=sys.stderr; validated by grep in acceptance criteria |
| T-02-02 | Tampering | post_message — kind field accepts arbitrary string | mitigate | validate_event() in store.py enforces kind enum=["message","position","dissent"]; ToolError raised before write if kind is invalid |
| T-02-03 | Elevation of Privilege | HARMONY_DB_PATH env var points to arbitrary path | accept | Phase 1 is internal-only, trusted agents; no authentication/path sandboxing in v1 scope (REQUIREMENTS.md: "trusted internal tools only") |
| T-02-04 | Tampering | payload_json field — unvalidated JSON string stored | mitigate | validate_event() allows only string-or-null type; format validation (valid JSON string) can be added in store.py write path if needed; accepted as low-risk for Phase 1 |
