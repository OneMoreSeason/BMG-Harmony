<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .mcp.json / config.toml → process spawn | The config files specify the Python executable and server script path; a compromised config could spawn a different process |
| harmony.sqlite → git | Store file must never be committed; .gitignore is the enforcement mechanism |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | harmony.sqlite in git history | mitigate | .gitignore includes .harmony/store/ and *.sqlite; .harmony/store/.gitkeep tracks the directory without the store file |
| T-03-02 | Tampering | .mcp.json Windows path escaping | mitigate | After write, file is validated as valid JSON before wiring; JSON parse failure caught before Claude Code loads it |
| T-03-03 | Denial of Service | MCP server not starting (HARMONY_DB_PATH not set) | mitigate | Server raises RuntimeError at lifespan start with clear message to stderr; agents see "Error starting server" rather than silent failure |
| T-03-SC | Tampering | No pip installs in this plan | accept | No new package installs; only config file edits and SETUP.md creation |
</threat_model>
