store.py exports:
  init_db(path: str) -> sqlite3.Connection
    Opens connection with check_same_thread=False, sets PRAGMA journal_mode=WAL,
    PRAGMA synchronous=NORMAL, executes DDL (CREATE TABLE IF NOT EXISTS for threads
    and events), installs the two append-only triggers. Returns open connection.

  write_event(con: sqlite3.Connection, event: dict) -> dict
    Validates event via validate_event(), auto-creates the thread with
    INSERT OR IGNORE INTO threads, inserts event row, commits, returns receipt dict:
    {event_id, thread_id, revision, timestamp, validation_status: "ok"}
    revision = rowid from the INSERT (cast to int).
    Raises ToolError on ValidationError or IntegrityError.

  read_thread(con: sqlite3.Connection, thread_id: str) -> list[dict]
    SELECT * FROM events WHERE thread_id=? ORDER BY rowid ASC.
    Returns list of row dicts (keys: event_id, thread_id, agent_id, kind,
    timestamp, content_md, payload_json). Empty list if thread not found.

  list_threads(con: sqlite3.Connection, state: str | None = None) -> list[dict]
    SELECT thread_id, slug, created_at, state FROM threads.
    Filters by state if provided. Returns list of row dicts.

schema.py exports:
  EVENT_SCHEMA: dict
    JSON Schema Draft7 object. required = [event_id, thread_id, agent_id, kind,
    timestamp, content_md]. kind enum = ["message", "position", "dissent"].
    payload_json type = ["string", "null"]. additionalProperties = False.

  validate_event(event: dict) -> None
    Calls _validator.validate(event). Raises jsonschema.ValidationError on failure.
    Never returns partial results.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent→store.py | Agent-supplied event dicts arrive via write_event(); could contain malformed data or SQL injection attempts in string fields |
| store.py→SQLite | All queries must use parameterized placeholders; never string-format SQL with user-supplied values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | write_event() — thread_id / agent_id string fields | mitigate | All SQLite queries use ? parameterized placeholders; no string formatting of user data into SQL |
| T-01-02 | Tampering | write_event() — payload_json field | mitigate | validate_event() checks payload_json type is string-or-null; caller must serialize to JSON string before passing; never store raw unvalidated dicts |
| T-01-03 | Tampering | events table — direct UPDATE/DELETE | mitigate | BEFORE UPDATE and BEFORE DELETE triggers with RAISE(ABORT) prevent mutation from any code path, not just application |
| T-01-04 | Information Disclosure | schema.sql in git | accept | Schema is intentionally public; no secrets or PII in DDL; store file is git-ignored (D-11, D-13) |
| T-01-SC | Tampering | pip install pytest | mitigate | pytest is in Package Legitimacy Audit table with slopcheck [OK]; installed via explicit version-pinned install; no other packages installed in this plan |
</threat_model>
